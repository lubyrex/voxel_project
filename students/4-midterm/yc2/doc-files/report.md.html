We will be looping over each pixel in the image we want to create, and combining the average of multiple rays (sent at slightly different angles) traced through that pixel to determine its radiance. We will do this with an inverted loop structure, doing all similar actions at the same time and storing the results in buffers to allow for the strategy of SIMD batch-processing multithreading. For the actual tracing part of our algorithm, we will be using Monte Carlo integration for calculating direct illumination (using probabilities determined by Importance Sampling). Indirect rays will be shosen using [Surfel::Scatter](http://graphics.cs.williams.edu/courses/cs371/f10/G3D/manual/class_g3_d_1_1_surfel.html#a49ece9dda77724eeb09ef8ea745a48f7) to give us optimal convergence. State any assumptions underlying that argument explicitly._ When we are calculating the true solution to the rendering equation for a surfel we are looking at the light emitted by that surfel plus the integral of all the light that is being reflected off of that surfel. However, as that is unrealistic to calculate, we approximate looking at all directions by using a Reimann Sum of a discrete number of randomly chosen directions, i.e. ∑L_i(X,L)∗f(ωi,ωo)∗|ωi⋅n|∗β(L,X). We control the span of this sum with two variables: - number of paths per pixel, which controls the number of random directions we are looking at to calculate the indirect light. - path length, with controls the number of times we look for indirect light. Therefore, assuming that as we randomly select directions to look for indirect light we are uniformly sampling the space, as paths per pixel and path length both approach infinity, our Reimann sum should become equivalent to the integral used to find the true solution to the rendering equation.
